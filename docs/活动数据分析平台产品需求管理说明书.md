## 一、需求背景

随着数字化运营的深入推进，活动营销已成为金融机构获取客户、促进转化的重要手段。当前活动运营过程中，存在以下问题：

1、**数据分散**：活动页面数据、推送数据、用户行为数据分散在多个系统，难以统一查看和分析；

2、**分析维度单一**：缺少从活动曝光到业务转化的全链路数据分析能力；

3、**决策支持不足**：运营人员难以快速获取活动效果的关键指标，无法及时调整运营策略；

为提升活动运营效率和数据驱动能力，需要建设统一的活动数据分析平台，实现活动效果的全面监测与分析。

---

## 二、需求目标

1、**统一数据看板**：整合活动页面分析、推送触达分析、用户行为分析、用户路径分析等多个模块数据，提供一站式活动数据分析视图；

2、**双视图切换**：支持"活动分析"和"任务分析"两种查看模式，满足不同角色的分析需求；

3、**全链路漏斗**：提供从页面曝光→推送触达→点击转化→业务激活的完整漏斗分析；

4、**业务转化追踪**：追踪激活客户数、激活千元户数、净新增资产、净创收等业务核心指标；

5、**数据明细导出**：支持多维度数据明细查看及CSV导出功能，便于深度分析和报告制作；

## 五、页面现状分析

### 5.1 活动分析页

活动分析视图是默认展示的数据分析页面，主要展示活动页面的流量情况和用户参与情况。

**包含模块：**

1、**页面展示分析**
   - 核心指标：活动页PV、活动页UV、目标用户数、参与用户数、目标完成率、活动进度
   - 页面漏斗图：展示用户访问页面的转化路径
   - PV来源占比：展示不同流量来源的占比分布

2、**推送触达分析**
   - 核心指标：触达客户数、点击链接数、推送成功数、推送失败数
   - 渠道引流占比：展示各推送渠道的客户分布
   - 推送链路漏斗：展示推送→送达→点击的转化情况

3、**用户行为分析**
   - 行为概览：时间窗口、总用户数
   - 行为分布图：各类用户行为的数量分布
   - 行为明细：各行为的详细数据列表

4、**数据明细表格**
   - 展示每日活动相关数据
   - 支持展开查看小时级别数据
   - 支持列配置和数据导出

### 5.2 任务分析页

任务分析视图聚焦于任务维度的业务转化情况，主要展示从触达到激活的业务成果。

**包含模块：**

1、**业务转化分析**
   - 核心指标：激活客户数、激活千元户数、净新增资产数、净创收、新增产品销售额
   - 激活条件说明：解释激活的定义
   - 转化漏斗图：触达→意向→激活的业务转化漏斗

2、**用户路径分析**
   - 路径漏斗：触达客户路径、意向客户路径、激活客户路径
   - 路径趋势图：各类型客户的转化趋势对比
   - 三类型客户路径对比：同页面多线路对比

3、**推送触达分析**
   - 同活动分析视图中的推送触达模块

4、**用户关键行为指标**
   - 可配置的关键行为指标卡片
   - 支持指标添加、删除操作
   - 展示指标名称、数值、单位、变化趋势

5、**数据明细表格**
   - 展示每日任务相关数据
   - 显示业务转化相关指标
   - 支持数据导出

## 六、产品详细方案

### 6.1 页面布局与结构

### 6.1.1 顶部导航栏

```
┌─────────────────────────────────────────────────────────────────────┐
│ ← │ 618理财狂欢节活动              日历  │ 数据更新时间：xxx         │
│   │                                        2024-06-01~06-30 │ 2024-06-18 14:30:00 │
└─────────────────────────────────────────────────────────────────────┘
```

**功能说明：**

1、**返回按钮**：左侧返回箭头按钮，点击返回上一级页面

2、**活动名称**：页面顶部左侧显示当前活动名称

3、**日期范围按钮**
   - 显示活动起止日期
   - 点击可选择日期范围（预留功能）
   - 默认显示配置的活动时间范围

4、**数据更新时间**：显示数据的最新更新时间点

**技术要求：**

- 导航栏固定在页面顶部，滚动时保持固定位置
- 活动名称过长时省略显示
- 日期范围按钮样式与设计稿保持一致

### 6.1.2 查看模式切换

```
┌─────────────────────────────────────────────────────────────────────┐
│ 查看模式：  ┌────────────┐  ┌────────────┐                          │
│            │  活动分析   │  │  任务分析   │                          │
│            └────────────┘  └────────────┘                          │
└─────────────────────────────────────────────────────────────────────┘
```

**功能说明：**

1、**切换按钮组**：两个按钮组成切换组件
   - "活动分析"：默认选中状态，蓝色背景白色文字
   - "任务分析"：未选中状态，白色背景蓝色文字

2、**切换逻辑**
   - 点击按钮切换查看模式
   - 切换后更新页面内容区域显示
   - 保持切换状态直至用户再次切换

**交互要求：**

- 按钮切换时有平滑的过渡效果
- 切换响应及时，内容区域无闪烁
- 当前选中状态有明显视觉区分

### 6.2 数据看板模块说明

### 6.2.1 页面展示分析

**位置**：活动分析视图，第一模块

**功能说明**：

1、**核心指标卡片（6个）**：

| 指标名称 | 单位 | 指标说明 |
|----------|------|----------|
| 活动页PV | 次 | 多会场PV汇总 |
| 活动页UV | 设备号 | 多会场UV集合去重 |
| 目标用户数 | 设备号 | CAMS目标看板填写的设备号数 |
| 参与用户数 | 设备号 | 完成/参与某事件（由业务定义） |
| 目标完成率 | % | UV / 目标用户数 × 100% |
| 活动进度 | % | (当前日期-开始日) / (结束日-开始日) |

2、**页面漏斗图**：

- 展示页面访问的转化路径
- 每个节点显示：页面名称、PV数、UV数、转化率
- 进度条宽度对应转化率百分比

3、**PV来源占比**：

- 饼图展示各流量来源的PV占比
- 图例显示来源名称、PV数、占比

### 6.2.2 推送触达分析

**位置**：活动分析视图，第二模块；任务分析视图，共用此模块

**功能说明**：

1、**核心指标卡片（4个）**：

| 指标名称 | 单位 | 指标说明 |
|----------|------|----------|
| 触达客户数 | 客户号 | 推送触达的客户数，去重 |
| 点击链接数 | 客户号 | 通过短链点击的客户数 |
| 推送成功数 | 次数 | 各渠道推送成功送达数 |
| 推送失败数 | 次数 | 各渠道推送失败数 |

2、**渠道引流占比**：

- 饼图展示各推送渠道触达的客户占比
- 渠道包括：短信、微信、APP推送、富信、AIM短信
- 部分渠道标注"无回执"状态

3、**推送链路漏斗**：

- 展示推送发送→送达成功→点击链接的转化漏斗
- 每个节点显示数量和转化率

### 6.2.3 用户行为分析

**位置**：活动分析视图，第三模块

**功能说明**：

1、**行为概览**：

- 展示数据时间窗口（如"7日"）
- 展示参与总用户数

2、**行为分布图**：

- 横向条形图展示各行为类型的用户数
- 条形颜色根据趋势变化显示（上涨为蓝色，下跌为红色）

3、**行为明细**：

- 表格形式展示各行为详情
- 包含行为名称、用户数、占比、趋势变化

### 6.2.4 用户路径分析

**位置**：任务分析视图，第二模块

**功能说明**：

1、**客户类型切换**：

- 三个按钮切换：触达客户、意向客户、激活客户
- 切换后更新路径漏斗图和趋势图

2、**路径漏斗**：

- 展示选定类型客户的转化路径节点
- 每个节点显示节点名称、用户数、占比

3、**路径趋势图**：

- 折线图展示当前选中类型的路径转化趋势

4、**三类型客户路径对比**：

- 同一图表中展示三条折线
- 分别代表触达客户、意向客户、激活客户的路径

### 6.2.5 业务转化分析

**位置**：任务分析视图，第一模块

**功能说明**：

1、**核心指标卡片（5个）**：

| 指标名称 | 单位 | 格式化说明 |
|----------|------|------------|
| 激活客户数 | 客户号 | 保留千分位分隔符 |
| 激活千元户数 | 户 | 保留千分位分隔符 |
| 净新增资产数 | 元 | ≥1亿显示X.XX亿 |
| 净创收 | 元 | ≥1万显示X万 |
| 新增产品销售额 | 元 | ≥1万显示X万 |

2、**激活条件说明**：

- 解释激活客户的定义

3、**转化漏斗图**：

- 展示业务转化漏斗：触达客户→意向客户→激活客户

4、**口径说明提示**：

- 醒目标注活动UV和任务侧指标的口径差异

### 6.2.6 用户关键行为指标

**位置**：任务分析视图，第三模块

**功能说明**：

1、**指标卡片**：

- 展示业务定义的关键行为指标
- 每个卡片显示：指标名称、当前值、单位、变化趋势

2、**指标操作**：

- 支持添加新指标（按钮操作）
- 支持删除指标（配置权限控制）
- 支持编辑指标（配置权限控制）

3、**交互说明**：

- 变化趋势为正数显示绿色向上箭头和百分比
- 变化趋势为负数显示红色向下箭头和百分比

### 6.2.7 数据明细表格

**位置**：活动分析视图和任务分析视图，均在最后模块

**功能说明**：

1、**日报数据**：

- 表格展示每日汇总数据
- 列根据视图模式不同而变化

2、**小时报数据**：

- 部分日期支持展开查看小时级别数据
- 点击展开按钮显示24小时明细

3、**列配置功能**：

- 鼠标悬停"显示列"按钮显示列配置面板
- 可勾选/取消勾选控制列的显示隐藏

4、**CSV导出功能**：

- 点击"CSV下载"按钮导出当前数据
- 导出文件名为"{viewMode}_data.csv"

### 6.3 交互说明

### 6.3.1 时间范围选择

1、顶部日期范围按钮点击后弹出日期选择器（预留）

2、选择日期后刷新页面各模块数据

3、默认显示当前活动时间范围

### 6.3.2 任务切换

1、任务列表下拉选择（预留）

2、切换任务后更新各模块数据

3、任务状态展示：运行中（绿色）、待开始（灰色）、已结束（灰色）

### 6.3.3 数据导出

1、点击"CSV下载"按钮触发导出

2、导出流程：
   - 获取当前显示列配置
   - 生成CSV格式数据
   - 添加BOM头（UTF-8编码标识）
   - 浏览器下载弹出

3、导出文件命名：
   - 活动分析视图：activity_data.csv
   - 任务分析视图：task_data.csv

## 七、数据指标说明

### 7.1 口径定义

| 指标类型 | 指标名称 | 统计口径 | 数据来源 |
|----------|----------|----------|----------|
| 页面指标 | 活动页PV | 多会场页面浏览次数汇总 | 埋点系统 |
| | 活动页UV | 多会场访问用户设备号去重数 | 埋点系统 |
| | 目标用户数 | CAMS目标看板填写的设备号数 | CAMS系统 |
| | 参与用户数 | 完成或参与预设事件的设备号数 | 埋点系统 |
| 任务指标 | 触达客户数 | 各渠道推送触达的客户号去重数 | 推送系统 |
| | 点击链接数 | 通过短链点击的客户号数 | 推送系统 |
| | 推送成功数 | 各渠道推送成功送达次数 | 推送系统 |
| | 推送失败数 | 各渠道推送失败次数 | 推送系统 |
| 转化指标 | 意向客户数 | 外呼系统有意向客户 + 推送点击链接客户的并集 | 业务系统 |
| | 激活客户数 | 完成激活行为的客户号数 | 业务系统 |
| 业务指标 | 激活千元户数 | 激活且资产≥1000元的户数 | 业务系统 |
| | 净新增资产 | 新增资产 - 流失资产，单位：元 | 业务系统 |
| | 净创收 | 业务带来的净收入，单位：元 | 业务系统 |

### 7.2 指标计算规则

1、**目标完成率** = 活动页UV / 目标用户数 × 100%

2、**活动进度** = (当前日期 - 开始日期) / (结束日期 - 开始日期) × 100%

3、**页面转化率** = 当前页面PV / 上一级页面PV × 100%

4、**渠道占比** = 该渠道客户数 / 总触达客户数 × 100%

5、**推送漏斗转化率** = 当前节点数量 / 上一节点数量 × 100%

6、**业务漏斗转化率** = 当前阶段客户数 / 触达客户数 × 100%

7、**净新增资产格式化规则**：
   - ≥100,000,000：显示为"X.XX亿"
   - ≥10,000：显示为"X万"
   - 其他：保留千分位分隔符

### 7.3 数据埋点要求

1、**页面浏览埋点**：每次页面访问触发，包含页面标识、用户标识、时间戳

2、**用户行为埋点**：预设行为事件触发，包含行为类型、用户标识、时间戳

3、**推送事件埋点**：推送发送、送达、点击事件分别触发

4、**业务行为埋点**：激活、购买等业务行为触发

## 八、非功能需求

### 8.1 性能要求

1、**页面加载时间**：首屏加载时间≤3秒（网络正常情况）

2、**数据刷新**：手动刷新到数据展示完成≤2秒

3、**图表渲染**：图表数据更新到渲染完成≤500ms

4、**用户操作响应**：按钮点击等交互响应≤100ms

### 8.2 安全要求

1、**数据权限**：根据用户权限展示可访问的数据范围

2、**敏感信息**：敏感数据（如客户号）脱敏显示

3、**操作日志**：关键操作记录操作日志

4、**数据传输**：数据请求使用HTTPS协议

### 8.3 兼容性要求

1、**浏览器兼容**：支持Chrome、Safari、Firefox、Edge等现代浏览器

2、**移动端适配**：支持Pad端访问，主要功能可用

3、**响应式布局**：页面可自适应不同屏幕宽度

4、**视网膜屏幕**：图表在高分辨率屏幕下清晰显示
